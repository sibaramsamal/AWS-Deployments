# 🚀 FULL NGINX + DOMAIN + SSL SETUP (REALISTIC FLOW)
## 🧱 STEP 1 — Install Nginx
```
sudo apt update
sudo apt install nginx -y
```
What’s happening
- Installs Nginx web server
- Starts service automatically  

To verify: `sudo systemctl status nginx`
👉 You should see active (running)

## 🌐 STEP 2 — Point Domain to Server (DNS)
In your domain provider (GoDaddy): Profile → browse all your domains → DNS → look for all entries

✅ Correct setup:
```
Type	    Name	    Value
A	        @	    YOUR_SERVER_IP
A	        dev	    YOUR_SERVER_IP
CNAME	    www	    yourdomain.com
```
🔴 Common possible mistake:  Don't let this entry present in your setup.
`A @ → WebsiteBuilder Site ❌` remove this entry.  
👉 This overrides your IP and sends traffic to GoDaddy

## 🔍 Verify DNS
```
dig yourdomain.com +short
👉 Must return: YOUR_SERVER_IP
```
## ⚙️ STEP 3 — Create Nginx Config
```
sudo nano /etc/nginx/sites-available/yourdomain.com
```
### Basic config (HTTP only)
```
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }

    location / {
        proxy_pass http://localhost:YOUR_APP_PORT;
        proxy_http_version 1.1;

        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

What’s happening  
- listen 80 → HTTP traffic
- server_name → domain binding
- proxy_pass → forwards to your app (Node, React, etc.)
- well-known → required for SSL validation

## 🔗 STEP 4 — Enable Site (VERY IMPORTANT)
```
sudo ln -s /etc/nginx/sites-available/yourdomain.com /etc/nginx/sites-enabled/
```
What this does  
- Activates your config
- Nginx only reads sites-enabled/

🔴 Mistake you could make
Forget this → site won’t load

## 🔁 STEP 5 — Restart Nginx
```
sudo nginx -t
sudo systemctl restart nginx
```
What this checks  
- nginx -t → validates config
- restart → applies changes

## 🔐 STEP 6 — Install SSL (Certbot)
`sudo apt install certbot python3-certbot-nginx -y`

🔴 Critical mistake may do
Don't add these lines manually
```
listen 443 ssl;
ssl_certificate ......
👉 Before certificate exists → ❌ Nginx crashes

✅ Correct approach
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```
What Certbot does  
- Verifies domain ownership
- Creates SSL cert
- Injects SSL config into Nginx
- Adds HTTP → HTTPS redirect

## 🔁 STEP 7 — Verify SSL
`curl -I https://yourdomain.com`

# Practical Implementation
### step 1
```
sudo nano  /etc/nginx/sites-available/queuefree.com
```
### step 2 (Dev + Main config)
```
# =========================
# PROD (queuefree.com)
# =========================
server {
    server_name queuefree.com www.queuefree.com;

    location / {
        proxy_pass http://localhost:5001;
        proxy_http_version 1.1;

        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    listen 443 ssl;
    ssl_certificate /etc/letsencrypt/live/queuefree.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/queuefree.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;
}

# Redirect HTTP → HTTPS (PROD)
server {
    listen 80;
    server_name queuefree.com www.queuefree.com;
    return 301 https://$host$request_uri;
}


# =========================
# DEV (dev.queuefree.com)
# =========================
server {
    server_name dev.queuefree.com;

    location / {
        proxy_pass http://localhost:5000;   # dev app port
        proxy_http_version 1.1;

        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    listen 443 ssl;
    ssl_certificate /etc/letsencrypt/live/queuefree.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/queuefree.com/privkey.pem;
}

# Redirect HTTP → HTTPS (DEV)
server {
    listen 80;
    server_name dev.queuefree.com;
    return 301 https://$host$request_uri;
}
```
